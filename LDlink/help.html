Understanding Linkage Disequilibrium

What is linkage disequilibrium? Perhaps it is best to start with linkage equilibrium. Linkage equilibrium exists when alleles from two different genetic variants are inherited independently of each other. Recombination during gamete formation results in a mixing of parental alleles such that the inheritance of variants follows probabilistic patterns governed by population allele frequencies. The vast majority of genetic variants on a chromosome are in linkage disequilibrium. Variants in linkage disequilibrium are not considered linked.

Linkage disequilibrium is present when alleles from two nearby genetic variants are inherited in a non-random, linked fashion. This linked mode of inheritance results from genetic variants in close proximity being less likely to be separated by a recombination event and thus alleles of the variants are more commonly inherited together than expected. Alleles of variants in linkage disequilibrium are correlated; with the degree of correlation generally greater in magnitude the closer the variants are in physical distance. Measures of linkage disequilibrium include D' and R square.

A haplotype is a cluster of genetic variants that are inherited together. Humans are diploid; having maternal and paternal copies of each autosomal chromosome. Each chromosomal copy is organized into segments of high linkage disequilibrium, called haplotype "blocks". Due to unique population histories and differences in variant allele frequencies, haplotype structure tends to be population specific. Although haplotypes are essential for calculating measures of linkage disequilibrium, haplotypes are seldom directly observed. Advanced chromosome phasing techniques are often necessary to infer individual haplotypes.

Data Sources

To investigate patterns of linkage disequilibrium, LDlink focuses on a specific class of genetic variants called single nucleotide polymorphisms (SNPs). Every module of LDlink requires the entry of at least one SNP as identified by an RefSNP number (RS number). RS numbers have been assigned by dbSNP and are well-curated identifiers that follow the format "rs" followed by 1 to 8 numbers. The current implementation of LDlink references dbSNP build 141 and only accepts input for bi-allelic, autosomal SNPs.

Publically available reference haplotypes from Phase 3 of the 1000 Genomes Project are used by LDlink to calculate population-specific measures of linkage disequilibrium. Haplotypes are available for continental populations (ex: European, African, and Admixed American) and sub-populations (ex: Finnish, Gambian, and Peruvian). All LDlink modules require the selection of at least one 1000 Genomes Project sub-population, but several sub-populations can be selected simultaneously. Available haplotypes vary by sub-population based on sample size.

Calculations

LDlink modules report the following measures of linkage disequilibrium: D prime, R square, and goodness-of-fit statistics. Below is a brief description of each measure.

D prime (D') - an indicator of allelic segregation for two genetic variants. D' values range from 0 to 1 with higher values indicating tight linkage of alleles. A D' value of 0 indicates no linkage of alleles. A D' value of 1 indicates at least one expected haplotype combination is not observed.

R square (R2) - a measure of correlation of alleles for two genetic variants. R2 values range from 0 to 1 with higher values indicating a higher degree of correlation. An R2 value of 0 indicates alleles are independent, whereas an R2 value of 1 indicates an allele of one variant perfectly predicts an allele of another variant. R2 is sensitive to allele frequency.

Goodness of Fit (Χ2 and p-value) - statistical test testing whether observed haplotype counts follow frequencies expected from variant allele frequencies. High chi-square statistics and low p-values are evidence that haplotype counts deviate from expected values and suggest linkage disequilibrium may be present.

Modules

LDlink consists of four modules: LDhap, LDmatrix, LDpair, and LDproxy. Each module can be accessed by clicking on the desired tab at the top of each LDlink page. Below is a description of each module, the required user input, and an explanation of the returned output.

LDhap - Calculate population-specific haplotype frequencies of all haplotypes observed for a list of query SNPs.

Input:

(1) List of RS numbers - this can either be entered one per line in the text entry box or uploaded as a file that contains a list of RS numbers in the first column. A maximum of 30 SNP RS numbers are permitted. All input SNPs must be on the same chromosome and match a bi-allelic, autosomal SNP.

(2) Reference population(s) - selected from the drop down menu. At least one 1000 Genomes Project sub-population is required, but more than one may be selected.

Output:

(1) Table of observed haplotypes - haplotypes are displayed vertically and sorted by observed frequency in the selected query sub-population. SNP genotypes are reported in rows and follow the same order as entered in the query input. Links are available to dbSNP RS numbers and coordinates in the UCSC Genome Browser.

(2) SNPs file download - download a file listing the order of the queried SNPs. SNPs are in the same order as entered in the query and are not ordered by genomic location (unless ordered by position in the query). This file is used as a key for the order of genotypes in the haplotype file. Only bi-allelic SNPs with matching dbSNP RS numbers are included. All others are filtered out.

(3) Haplotype file download - download a file of all observed haplotypes. Haplotype genotypes are in the order of the above SNPs file.

LDmatrix - Create interactive heat maps of pairwise linkage disequilibrium statistics.

Input:

(1) List of RS numbers - this can either be entered one per line in the text entry box or uploaded as a file that contains a list of RS numbers in the first column. A maximum of 100 SNP RS numbers are permitted. All input SNPs must be on the same chromosome and match a bi-allelic, autosomal SNP.

(2) Reference population(s) - selected from the drop down menu. At least one 1000 Genomes Project sub-population is required, but more than one may be selected.

Output:

(1) Interactive heat map - a square plot with dimensions equal to the number of query SNPs that match a dbSNP RS number. Hovering over the plot with the mouse will display pairwise LD metrics for the respective row and column SNPs. SNPs are ordered the same way as the RS numbers in the query and are not ordered by genomic coordinates (unless ordered by position in the query).

(2) D prime download - download a file of all pairwise D' statistics.

(3) R square download - download a file of all pairwise R2 statistics.

LDpair - Investigate correlated alleles for SNPs in high LD.

Input:

(1) SNP1 RS number - RS number for query SNP1. RS number must match a bi-allelic, autosomal SNP.

(2) SNP2 RS number - RS number for query SNP2. RS number must match a bi-allelic, autosomal SNP.

(3) Reference population(s) - selected from the drop down menu. At least one 1000 Genomes Project sub-population is required, but more than one may be selected.

Output:

(1) Two-by-two table - contingency table displaying haplotype counts and allele frequencies for the two query SNPs. External links are available to dbSNP RS numbers and coordinates in the UCSC Genome Browser.

(2) Haplotypes - haplotype genotypes, counts, and frequencies.

(3) Statistics - calculated metrics of linkage disequilibrium including: D prime (D'), R square (R2), and goodness-of-fit (Chi-square and p-value). Goodness-of-fit tests for deviations of expected haplotype frequencies based on allele frequencies.

(4) Correlated Alleles - alleles that are correlated if linkage disequilibrium is present (R2 > 0.1). If linkage equilibrium, no alleles are reported.

LDproxy - Interactively explore proxy and putatively functional SNPs for a query SNP. Based on the selected query populations, this module may take some time to run.

Input:

(1) SNP RS number - RS number for query SNP. RS number must match a bi-allelic, autosomal SNP.

(3) Reference population(s) - selected from the drop down menu. At least one 1000 Genomes Project sub-population is required, but more than one may be selected.

Output:

(1) Interactive plot - interactive plot of query SNP and all bi-allelic dbSNP SNPs plus or minus 500 kilobases of the query SNP. X axis is the chromosomal coordinates and the Y axis is the pairwise R2 value with the query SNP. Each point represents a proxy SNP and is colored based on function, sized based on minor allele frequency, and labeled based on regulatory potential. Hovering over the point will display detailed information on the query and proxy SNPs.

(2) UCSC link - external link to the plotted region (query SNP -/+ 500 Kb) in the UCSC Genome Browser. This is useful for exploring nearby genes and regulatory elements in the region.

(3) Table of top 10 proxies - the ten SNPs with the highest R2 values and closest distance to the query SNP are displayed. External links lead to the SNP RS number in dbSNP, coordinates in the UCSC Genome Browser, and regulatory information (if any) in RegulomeDB.

(4) Download all proxy SNPs - download a file with information on all SNPs -/+ 500 Kb of the query SNP with a pairwise R2 value greater than 0.1.

Frequently Asked Questions

Why is my SNP RS number bringing up an error?

LDlink modules only accept input for SNP RS numbers that are bi-allelic and autosomal. Ensure your query SNP has A/C, A/G, A/T, C/G, C/T, or G/T alleles. RS numbers for insertions or deletions (i.e. "indels") are not currently accepted as input. If a SNP you believe is bi-allelic is not accepted, check dbSNP to ensure there are only two alleles for the SNP. Even if dbSNP only reports one observation of a SNP being tri- or multi-allelic, the SNP is not considered valid input under the current implementation of LDlink.

What 1000 Genomes population should I select for my LDlink query?

Choosing the correct reference 1000 Genomes Project population is essential for comparability of results from LDlink to your population of interest. In general, try to select the sub-population that best matches the ancestry of your study population. While LDlink allows for multiple 1000 Genomes sub-populations to be selected simultaneously, it is recommended to first investigate the patterns of linkage disequilibrium in the query region of each sub-population before considering whether to combine sub-populations in a query.

What is the estimated running time for an LDproxy query?

Running time varies greatly for LDproxy ranging from a few seconds to approximately one minute. For each query, LDproxy actively calculates all LD metrics based on user input. A variety of factors affect overall running time including: (1) the number of haplotypes available for the sub-population, (2) the number of dbSNP variants in the region queried, (3) the current utilization of the LDlink server, and (4) the download speed of your internet connection. In general, the best way to speed up an LDlink query is to only select one sub-population per query. Queries that include multiple sub-populations or ancestral groups take much longer than single sub-population queries.

How do I save output files?

Output from a variety of LDlink modules (LDhap, LDmatrix, and LDproxy) is available for download. To save output either (1) right click or long press on the link and select the "Save Link As…" option or (2) open the link in a new browser window and paste the contents to a new file.

Can I save the plots generated in LDmatrix or LDproxy?

Yes. Click the Save icon on the top right of the plot. A preview window will appear. Right click or long press on the preview image, select "Save As…", and choose a filename for the plot image.

What is the maximum number of SNPs LDhap and LDmatrix can accept as input?

LDhap has a limit of 30 SNPs and LDmatirx has a limit of 100 SNPs. These limits were selected to optimize query speed and visualization of results.

Why are the SNPs in LDhap and LDmatrix output not ordered by chromosomal position?

SNPs in LDhap and LDmatrix output are ordered based on user input. If the user inputs the SNP RS numbers in chromosomal order the results will be in chromosomal order. However, if the SNPs are entered in a different order, the output will reflect this order. In general, it is best to input the SNP RS numbers in the order they appear on a chromosome, but the option is available for other orderings is the user so chooses.

Why does my computer seem bogged down by LDlink?

The interactive plots produced by LDmatrix and LDproxy require a sizeable amount of data and java script to be loaded into the web browser. On some computers this results in a notable lag in performance. To improve performance, close the web page or simply refresh the LDlink webpage to clear the data from memory. Clearing the data will remove all interactive plots.
